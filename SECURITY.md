# Security Policy

Please do not report security issues in public issues.

If you discover a vulnerability in this project:

- use GitHub private vulnerability reporting if it is enabled for the repository
- otherwise contact the maintainer privately before public disclosure

When reporting a security issue, include:

- the affected route, file, or feature
- reproduction steps
- expected impact
- any suggested mitigation

The project aims to acknowledge credible reports quickly and patch confirmed issues responsibly.
