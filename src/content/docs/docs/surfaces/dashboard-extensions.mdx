Dashboard surfaces exist for admin-facing work.

If the site owner needs a place to configure, review, manage, or decide, dashboard is usually the right
home.

<MentalModelDiagram focus="dashboard" />

## Dashboard pages

Dashboard pages are the workhorse admin surface.

Use them for:

- settings
- onboarding
- management flows
- reports
- CRUD-heavy tools

The rule of thumb is simple: if the user needs room to think, compare, edit, or manage, give the task
its own page.

## Dashboard modals

Dashboard modals are for shorter, tighter actions.

Use them for:

- confirmations
- quick edits
- compact forms
- focused detail views

If the task feels like a side action rather than a destination, a modal is often enough.

## Dashboard plugins and menu plugins

These are more specialized extension points inside existing Wix dashboard flows or menus.

They are powerful, but they are not the best first lesson for most builders.

Learn pages well first. Add plugin complexity only when your product genuinely needs to attach itself
to an existing Wix-owned surface.

## UI expectations

The dashboard is inside a Wix-controlled admin shell. Let that reality guide the interface.

Strong dashboard UI usually means:

- clear labels
- steady spacing
- practical density
- standard controls
- calm information hierarchy

Trying to make dashboard UI feel like a flashy marketing site usually creates friction instead of polish.
