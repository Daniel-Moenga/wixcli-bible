The Wix CLI workflow is easier to remember when you stop treating commands as isolated facts.

## The full loop

Study this until it feels automatic:

<Steps>

1. **Create** -- Scaffold the project with `npm create @wix/new@latest app`.

2. **Login** -- Authenticate your terminal with `wix login`.

3. **Dev** -- Start the local development server with `wix dev`.

4. **Generate** -- Add new extensions (dashboard pages, widgets, APIs) with `wix generate`.

5. **Code** -- Build the feature in your editor.

6. **Build** -- Compile project assets with `wix build`.

7. **Preview** -- Upload a hosted build for testing with `wix preview`.

8. **Release** -- Register extensions with Wix and go live with `wix release`.

</Steps>

<WorkflowLoopDiagram />

Not every session needs every step, but almost every meaningful delivery fits inside this sequence.

## What most daily work looks like

<Tabs>
	<TabItem label="Daily loop">
		Most real sessions are shorter than the full loop:

		```bash
		wix dev
		# code, test, repeat
		```

		You stay in `dev` mode, make changes, see them reflected. The later steps matter when you're ready to share or ship.
	</TabItem>
	<TabItem label="Full release">
		When you're shipping to production, you need the full sequence:

		```bash
		wix build
		wix preview   # test the hosted build
		wix release    # register everything with Wix
		```

		Don't skip `build` before `preview` or `release` -- they need compiled assets.
	</TabItem>
</Tabs>

## Preview vs release

:::tip[The mental shortcut]
**Preview** = "Can other people test this build?"
**Release** = "Has Wix fully recognized and registered the pieces?"
:::

This distinction matters most when:

- A site widget feels half-wired
- An embedded script behaves differently than expected
- A generated extension exists locally but doesn't show up inside Wix

When something looks strange after a successful preview, `release` is usually the next thing to check.

## The operational mindset

Think of these stages as different questions:

| Stage | Question it answers |
|-------|-------------------|
| `dev` | Can I build and iterate locally? |
| `preview` | Can other people test this build? |
| `release` | Has Wix fully recognized the extensions? |

That mental separation prevents a lot of confusion.
