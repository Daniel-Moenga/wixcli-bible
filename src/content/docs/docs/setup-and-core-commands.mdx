The Wix CLI becomes much easier once you learn the real command set instead of copying random terminal snippets from older tutorials.

## Prerequisites

<Steps>

1. **Install Node.js** -- Use a current LTS version. The CLI needs it.

2. **Install Git** -- For version control and project collaboration.

3. **Create a Wix account** -- You'll need one to authenticate and connect your project.

4. **Open a terminal** -- Any terminal works. You don't need advanced shell skills, just the ability to navigate folders and run commands.

</Steps>

## Create a project

<Tabs>
	<TabItem label="App project">
		The most common path. Creates a Wix app with dashboard pages, site extensions, and backend APIs.

		```bash
		npm create @wix/new@latest app
		```

		This scaffolds the full project structure with extension registration already wired.
	</TabItem>
	<TabItem label="Headless project">
		Use this only when you know your frontend lives outside Wix.

		```bash
		npm create @wix/new@latest headless
		```

		Headless projects connect to Wix APIs but don't use the Wix admin shell for UI.
	</TabItem>
</Tabs>

After scaffolding, your project looks like this:

<FileTree>

- src/
  - dashboard/
    - pages/ Dashboard page extensions
  - backend/ Server-side logic
  - **extensions.ts** Extension registration
- public/ Static assets
- .wix/ Wix metadata
- wix.config.json Project config
- package.json Dependencies
- tsconfig.json TypeScript config

</FileTree>

## The commands to learn first

These seven commands cover 90% of what you'll do:

### `wix login` <Badge text="Auth" variant="note" />

Authenticate before running commands that need account access.

```bash
wix login
```

### `wix whoami` <Badge text="Auth" variant="note" />

Check which account is active. Use this when auth feels wrong or after switching accounts.

```bash
wix whoami
```

### `wix dev` <Badge text="Dev" variant="success" />

Start the local development server. This is your most common command.

```bash
wix dev
```

### `wix generate` <Badge text="Dev" variant="success" />

Add new extensions to your project -- dashboard pages, widgets, backend APIs.

```bash
wix generate
```

### `wix build` <Badge text="Deploy" variant="tip" />

Compile project assets before preview or release.

```bash
wix build
```

### `wix preview` <Badge text="Deploy" variant="tip" />

Create a hosted preview build for testing and sharing.

```bash
wix build && wix preview
```

### `wix release` <Badge text="Deploy" variant="tip" />

Release the project and handle extension registration with Wix.

```bash
wix build && wix release
```

## More commands

| Command | What it does | When to use it |
|---------|-------------|---------------|
| `wix logout` | Logs you out | Switching accounts or cleaning up a shared machine |
| `wix env set` | Sets an environment variable | Configuring secrets or feature flags |
| `wix env pull` | Pulls remote env vars to local | Onboarding a new machine |
| `wix env remove` | Removes an environment variable | Cleaning up stale config |
| `wix skills add` | Adds AI tool integrations | Setting up AI-assisted development |
| `wix telemetry` | Toggles anonymous telemetry | Opting in or out of usage data |
| `wix connect` | Connects headless projects to Vibe | Linking to the visual Vibe editor |

:::note
`skills add`, `telemetry`, and `connect` are newer additions. If your CLI doesn't recognize them, update with `npm i -g @wix/cli@latest`.
:::
