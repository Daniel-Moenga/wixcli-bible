Wix CLI projects stop feeling strange once you separate your real code from the generated machinery.

## The project at a glance

<FileTree>

- src/
  - dashboard/
    - pages/ Your dashboard page extensions
  - site/
    - widgets/ Your site widget extensions
    - embedded-scripts/ Embedded scripts
  - backend/ Server-side logic and APIs
  - **extensions.ts** The registration file -- wires extensions to Wix
- public/ Static assets (logos, images, files served directly)
- .wix/ Local Wix metadata (don't edit by hand)
- .astro/ Generated framework artifacts
- wix.config.json Project metadata (edit carefully)
- astro.config.mjs Framework configuration
- tsconfig.json TypeScript settings
- package.json Dependencies and scripts

</FileTree>

## What you edit vs what's generated

<Tabs>
	<TabItem label="Your code">
		These are the folders that contain your actual product logic:

		- **`src/dashboard/`** -- Dashboard pages, modals, settings screens
		- **`src/site/`** -- Widgets, embedded scripts, anything visitors see
		- **`src/backend/`** -- Validation, API calls, secrets, event handlers
		- **`src/extensions.ts`** -- The registration file that tells Wix what you built
		- **`public/`** -- Static assets served directly

		This is where your time should go.
	</TabItem>
	<TabItem label="Generated files">
		These are created and maintained by the framework:

		- **`.wix/`** -- Local Wix metadata and support material. Recognize it, don't edit it.
		- **`.astro/`** -- Generated framework artifacts. Useful to know about, rarely edited by hand.
		- **`wix.config.json`** -- Project metadata. Don't change this unless you know exactly why.
		- **`node_modules/`** -- Dependencies. Managed by your package manager.

		You can safely ignore these until something specific requires attention.
	</TabItem>
</Tabs>

## The registration file

`src/extensions.ts` is the most important file to understand conceptually.

It tells Wix which extensions your project provides:

```ts
app().use(...extensionBuilders)
```

The existence of code is not enough by itself. Wix still needs to know that the extension is registered as part of the app. This is why the generated structure is worth respecting early on.

:::caution
Reorganize your project only after you understand the wiring you're changing. Moving extension folders without updating the registration chain will silently break things.
:::

## Generated extension folders

`wix generate` creates default folder paths based on the extension type you chose.

Beginner-safe advice:

1. Keep the generated shape at first
2. Move things later only with intention
3. Update references carefully when you reorganize

The goal isn't to preserve scaffolding forever. The goal is to avoid breaking the registration chain before you understand it.
