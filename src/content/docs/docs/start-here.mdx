If you only read one page before touching code, read this one.

## What the current Wix CLI is

The current Wix CLI is the unified command-line workflow for creating, developing, previewing, and
releasing Wix app projects and Wix Headless projects.

The most important mindset shift is this:

> Think workflow, not scaffolding.

The CLI is not just a project generator. It is the system that carries you through project creation,
local development, extension generation, build output, preview URLs, release behavior, and environment
management.

## What this handbook covers first

This handbook is deliberately app-first.

That means it focuses on the surface area where most people get stuck:

- dashboard pages and modals
- site widgets and embedded scripts
- backend APIs and secure logic
- release behavior and extension registration
- environment variables, permissions, and self-hosting

Headless is still part of the CLI family, but it is not the best first learning path unless you already
know that your frontend will live outside Wix.

## What this handbook is not

This project is not a replacement for the official Wix docs.

Wix changes. Product teams rename things. commands evolve. release notes matter. The official docs stay
canonical. This handbook exists to make those docs easier to understand and easier to navigate.

## How to use it well

Use it in one of three modes:

1. Study mode: read the pages in order and build the model before you optimize for speed.
2. Build mode: keep the docs open while working and jump straight to setup, workflow, or the surface you are building.
3. Troubleshooting mode: use the guides, quick reference, and Ask AI Beta when something behaves differently from what you expected.

## The beginner-safe rule

If you are unsure which Wix documentation track applies to your project, verify that first.

One of the fastest ways to get lost is mixing:

- the current unified Wix CLI
- older Wix CLI for Apps material
- Wix CLI for Sites material

The handbook repeats that distinction on purpose because it causes real confusion in practice.

## What success looks like

After the next few pages, you should be able to answer these questions clearly:

- Which surface does this feature belong to?
- Am I building a Wix app project, a headless project, or looking at site-specific docs?
- Which commands matter in the normal loop?
- Why would preview differ from release?

If you can answer those, the rest of the platform becomes much easier.
