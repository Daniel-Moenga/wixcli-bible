# Style Guide

## Editorial

- Write for clarity before completeness.
- Prefer practical language over jargon.
- Keep Wix as the source of truth.
- Use short paragraphs and meaningful headings.
- Explain tradeoffs when the answer is not obvious.

## Technical

- Prefer current unified Wix CLI terminology.
- Cite official sources for behavior-sensitive claims.
- Keep examples small and realistic.
- Avoid mixing dashboard, site, and backend concerns in one explanation unless the architecture needs all three.

## Design

- Default dark mode stays first-class.
- Preserve the compact, rounded, low-shadow shell treatment.
- Use Wix Madefor Text as the primary family when technically available.
- Use Barlow as the supporting accent family.
